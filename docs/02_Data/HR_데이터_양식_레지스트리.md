# HR 데이터 양식 레지스트리 (Schema Registry)

> **목적**: raw 폴더에 들어오는 모든 HR 엑셀 파일의 양식 패턴을 기록.  
> 새 파일이 들어올 때 이 문서와 비교해서 **신규 양식 여부를 자동 판별**한다.  
> 신규 양식 감지 시 → `data/processed/bronze/unknown_schema_alert.json` 에 기록 + 이 문서 업데이트 요청.

---

## 분류 기준 (UK 컬럼 조합)

| 분류 코드 | 판별 조건 | 처리 방식 |
|-----------|-----------|-----------|
| `master` | 사번/사원코드 + (입사일 OR 퇴사일) | master_auto.csv에 통합 ingest |
| `reference_roster` | 이름만 + (입사일 OR 퇴사일), 사번 없음 | QA 검증용으로 별도 저장 |
| `aux_skill` | 스킬/기술스택/개발스텍 컬럼 존재 | aux_skill.csv |
| `aux_recruit` | 채용/지원자/경력기술서 컬럼 존재 | aux_recruit.csv |
| `aux_personalcard` | 주민등록번호 컬럼 OR 1인 폼 형태 | 주민번호 드랍 후 스킵 |
| `aux_pricing` | 단가/투입공수/매출/연봉 컬럼 존재 | aux_pricing.csv |
| `sensitive_drop` | 휴대폰 + 이메일 + 내선번호 조합 | 전체 드랍 (민감정보) |
| `org_chart` | 시트명/파일명에 "조직도" | 스킵 |
| `skip_guide` | 시트명이 "가이드"/"기준"/"설명" | 스킵 |
| `unknown` | 위 조건 미해당 | unknown_schema_alert 기록 |

## 등록된 양식 목록

### TYPE 1: master (인원현황 / 입퇴사 명단)

#### 1-1. 인원현황 기본형 (사번+주민번호+입퇴사일)
```
대표 파일: 2022년_인원현황.xlsx, 인력등급-20230921.xlsx
시트 구성: [퇴사자], [재직자]
핵심 컬럼: 구분, 사번, 이름, 직책/직급, 소속, 부서, 주민등록번호(→드랍), 입사일, 퇴사일
UK: 사번
민감정보: 주민등록번호 → ingest 시 자동 드랍
시트 판별: 시트명 "재직자" → master(재직), "퇴사자" → master(퇴사)
```

#### 1-2. 인원현황 등급기초형 (사번 없음 + 주민번호 + 입사일)
```
대표 파일: 2023년_인원현황-20231019-등급기초자료.xlsx
시트 구성: [퇴사자], [재직자], [프로그래머+엔지니어]
핵심 컬럼: 구분, 이름, 직급, 소속, 주민등록번호(→드랍), 입사일, 학위, 자격증
UK: 이름 + 입사일 (사번 없음 → 이름 기반 매칭)
민감정보: 주민등록번호 → 자동 드랍
시트 판별: 시트명으로 master 확정. "프로그래머+엔지니어" 시트도 master로 처리
```

#### 1-3. 퇴사자 명단 단순형 (사번+입퇴사일)
```
대표 파일: 2023년_퇴사자_명단_비밀번호_없음_.xlsx, 2024년_입사자__퇴사자_명단.xlsx
시트 구성: [2023년] 또는 [입사자], [퇴사자]
핵심 컬럼: 사원코드, 사원명, 입사일자, 퇴사일자
UK: 사원코드
민감정보: 없음
시트 판별: 시트명 "입사자" → join_date 기준, "퇴사자" → exit_date 기준
```

#### 1-4. 경력연차별 인원 복합형 (재직자/퇴사자/집계 혼합)
```
대표 파일: 아이엔소프트_구성원_경력연차별_인원_수-202400402.xlsx
시트 구성: [본부별 년차별 인원수], [프로그래머+엔지니어(써머리)], [아이엔소프트_직원목록], [재직자], [프로그래머+엔지니어], [퇴사자]
핵심 컬럼: 이름, 직책, 직급, 소속, 주민등록번호(→드랍), 입사일, 경력기준등급
UK: 이름 + 입사일
민감정보: 주민등록번호 → 자동 드랍
시트 판별:
  - "본부별 년차별 인원수" → skip (집계표)
  - "아이엔소프트_직원목록" → skip (참조용)
  - "재직자" / "퇴사자" / "프로그래머+엔지니어" → master
```

### TYPE 2: reference_roster (검증용 명단)

#### 2-1. 연도별 입사자/퇴사자 명단형
```
대표 파일: 2021_2024년_입사자__퇴사자_명단.xlsx
시트 구성: [2021년], [2022년], [2023년], [2024년]
핵심 컬럼: (번호), 입사자, 퇴사자 — 이름만 있음, 사번 없음
UK: 없음 (이름 기반)
용도: QA 교차검증용. master와 이름 비교해서 누락/불일치 탐지
판별 조건: 컬럼이 "입사자"/"퇴사자" 텍스트이고 사번 없음
```

### TYPE 3: aux_skill (스킬/역량 인벤토리)

#### 3-1. 기술스택 경험여부 매트릭스형
```
대표 파일: 구성원_기술스택__경험여부-20240112-__엔지니어_등급_.xlsx
시트 구성: [기술스택], [가이드], [아이엔소프트_직원목록]
핵심 컬럼: 이름, 직책, 직급, 소속, 경력기준등급 + 프로젝트/기술 경험 여부 (O/X 매트릭스)
UK: 이름
시트 판별:
  - "기술스택" → aux_skill (메인)
  - "아이엔소프트_직원목록" → 참조용 skip
  - "가이드" → skip
```

#### 3-2. 개인별 개발스텍 RnR형
```
대표 파일: 솔루션팀_개인별_개발스텍__RnR_및_업무_현황_20210219.xlsx
시트 구성: [Sheet1]
핵심 컬럼: 성명, 직급, 등급, 개발 분야, 업무 분야, 솔루션 파트, 개발스텍, 현재업무
UK: 성명
판별 조건: "개발스텍" OR "개발스택" OR "기술스택" 컬럼 존재
```

---

### TYPE 4: aux_recruit (채용 데이터)

#### 4-1. 채용 Skill Inventory 경력기술서형
```
대표 파일: IN_HR_채용_Skill_Inventory_성명.xlsx
시트 구성: [경력기술서]
특징: 개인별 경력기술서 형태. 표 구조가 아닐 수 있음
판별 조건: 시트명 "경력기술서" OR 파일명에 "채용"/"Skill Inventory"
처리: 컬럼 자동 추출 실패 시 aux_recruit로 분류 후 수동 확인 요청
```

### TYPE 5: aux_personalcard (평가/개인카드)

#### 5-1. 구성원 평가 등급 본부별 시트형
```
대표 파일: 구성원평가_등급_grade_-아이엔소프트-20241202_최종_.xlsx
시트 구성: [작성 기준], [임원급], [영업본부], [서비스사업본부], [클라우드네이티브본부], ...
핵심 컬럼: 구분, 등급(S/A/B/C/D), 성명, HR검토, 최종확정
특징: 시트마다 본부 다름. 헤더 위치 불규칙 (hdr=1~3)
판별 조건: 컬럼에 "등급" + "성명" 존재 AND "S","A","B","C","D" 값 패턴
처리: 등급 데이터만 추출. 주민번호 없음
```

### TYPE 6: aux_pricing (단가/연봉/프로젝트 실적)

#### 6-1. 연봉인상 시뮬레이션형
```
대표 파일: 연봉인상금액_Simu-20240112.xlsx
시트 구성: [아이엔소프트_직원목록]
핵심 컬럼: 이름, 직급, 소속, 등급 + 연봉 관련 수치
판별 조건: 파일명에 "연봉"/"Simu"/"인상" OR 컬럼에 연봉/급여 관련 수치
처리: aux_pricing으로 분류. 금액 컬럼 민감 처리
```

#### 6-2. 투입단가 등급분류형
```
대표 파일: 직원_투입단가_등급분류-20231020_내부용_.xlsx
시트 구성: [Sheet1]
핵심 컬럼: 이름, 직책, 직급, 소속, 경력기준등급, 자격증1~3, 총경력년수
판별 조건: "단가" OR "투입" 컬럼 존재
처리: aux_pricing
```

#### 6-3. 프로젝트 실적관리 템플릿형
```
대표 파일: 프로젝트_실적관리_Templete_2024년_1_0-20240306.xlsx
시트 구성: [프로젝트 실적관리], [투입인력(계획)], [투입인력(실적)], [매출매입경비], [아이엔소프트_직원목록], [실적관리 가이드]
핵심 컬럼: 소속본부, 성명, 직급, 내부단가, 계약단가, 투입공수, 투입일, 종료일
판별 조건: "단가" + "투입공수" 컬럼 동시 존재
시트 판별:
  - "투입인력(계획)" / "투입인력(실적)" / "매출매입경비" → aux_pricing
  - "아이엔소프트_직원목록" → skip (참조용)
  - "실적관리 가이드" → skip
```

### TYPE 7: sensitive_drop (민감정보 전체 드랍)

#### 7-1. 직원 연락망형
```
대표 파일: 아이엔소프트_직원연락망__23_12_01_.xlsx
시트 구성: [아이엔소프트_직원연락망]
민감 컬럼: 내선번호, 휴대폰, e-mail
판별 조건: "휴대폰" OR "핸드폰" OR "내선" + "e-mail" OR "이메일" 조합
처리: 전체 드랍. ingest 안 함. classified_skips["sensitive_drop"]에 기록
```

### TYPE 8: org_chart (조직도 → 스킵)

#### 8-1. 조직도 시각화형
```
대표 파일: 조직도-20230705.xlsx
시트 구성: [조직도 (병합)], [조직도 color], [조직도 (약식)]
특징: 병합셀 다수. 표 구조 아님. 시각적 레이아웃
판별 조건: 파일명 OR 시트명에 "조직도" 포함
처리: org_chart로 분류 후 스킵
```

### TYPE 9: aux_personalcard (개인 인사기록카드)

#### 9-1. 개인별 인사기록카드 폼형 (1인 1파일)
```
대표 파일: 2014_인사기록카드_김석진.xlsx, 2021년도 개인이력카드/ 폴더 내 파일들
시트 구성: [인사기록카드], [기타정보등록], [Sheet2]
특징: 세로 key-value 폼 형태. 1명 전체 정보. 표 구조 아님
민감 컬럼: 주민등록번호, 집주소, 핸드폰, 이메일, 급여계좌
판별 조건:
  - 파일명에 "인사기록카드" OR "개인이력카드" 포함
  - OR 행수 < 50 AND 컬럼수 > 20 (폼 형태 추정)
  - OR 주민등록번호 값 패턴 (######-#######) 감지
처리: aux_personalcard 분류 후 전체 드랍 (민감정보 과다)
```

## 공통 스킵 시트 패턴

아래 시트명이 감지되면 해당 시트는 ingest에서 제외:

```python
SKIP_SHEET_NAMES = [
    "가이드", "guide", "설명", "기준", "작성 기준",
    "실적관리 가이드", "Sheet2", "색인", "index",
    "아이엔소프트_직원목록",   # 참조용 목록 (중복 ingest 방지)
    "본부별 년차별 인원수",     # 집계표
]
```

## 신규 양식 감지 시 처리 절차

새 파일이 들어왔을 때 위 패턴과 **하나도 매칭되지 않으면**:

1. `data/processed/bronze/unknown_schema_alert.json` 에 아래 형식으로 기록
```json
{
  "file": "파일명.xlsx",
  "sheet": "시트명",
  "detected_cols": ["컬럼1", "컬럼2", "..."],
  "rows": 123,
  "reason": "어떤 패턴과도 매칭되지 않음",
  "timestamp": "2026-03-09T12:00:00"
}
```

2. ingest_report에 `"unknown_schema"` 키로 목록 포함

3. **이 문서 업데이트 명령어**:
```bash
# 새 파일 분석 후 레지스트리 업데이트
python main.py --update-schema-registry data/raw/새파일.xlsx
```

4. 업데이트 시 아래 항목을 이 문서에 추가:
   - TYPE 번호 및 이름
   - 대표 파일명
   - 시트 구성
   - 핵심 컬럼 (UK 포함)
   - 민감정보 여부
   - 판별 조건
   - 처리 방식
